# 06_BRIEF_TEMPLATE — Шаблон брифа задачи

Бриф самодостаточен: Executor работает без доступа к истории чатов.

```md
# BRIEF <ID> — <название>
**Тип:** verification | analysis | drafting | script | discovery
**Роль:** Executor (Sonnet) · **Дата:** · **Автор:** Architect

## Цель
<один абзац: что должно стать истинным после выполнения>

## Context-to-load (curl по SHA сессии)
- <литеральные raw-URL всех нужных файлов, включая конкретные фрагменты l3-external/md/>

## Входы и факты
<что уже известно/решено, с ссылками ADR-xxx, К-x>

## Шаги
1. ...

## Критерии приёмки
- <проверяемые условия; для verification: каждая P-xx получает статус + цитаты файл@SHA:строки>

## Return-this
<точный формат выхода: записи регистров / черновик задачи подрядчику / скрипт+ожидаемый вывод>
+ session-блок по 04_SESSION_PROTOCOL (обязателен)

## Вне scope
<что НЕ делать, даже если заметил>

## Эскалация
CONTEXT GAP / противоречие / соблазн додумать → стоп, поднять Architect. Не решать самому.
```

Discovery-бриф: то же, но Цель = добыть знание (вопрос подрядчику через Contractor Question Register, выгрузка, замер); Return-this = обогащение репо, не прод-артефакт.
