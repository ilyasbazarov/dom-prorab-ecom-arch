Fast Track

### **Fast Track API (Цены и Остатки)**

**Цель:** Обеспечить витрину (Битрикс) актуальными ценами и чистыми остатками с задержкой не более 5-10 минут, не перегружая при этом сервер 1С лишними вычислениями и дублями.

#### **1. Логика формирования очереди и отправки батчей (Пайплайн 1С)**

Подрядчик 1С должен реализовать асинхронный механизм "Регистрации изменений" (чтобы не тормозить работу кассиров в реальном времени).

- **Шаг 1: Триггер (Запись в очередь).** При любом проведении документа, меняющего остаток или цену товара, 1С *мгновенно* и "тихо" пишет в служебный регистр Очередь_Fast_Track строку: \[GUID, Склад_ID, Дата_Время\].

- **Шаг 2: Сборщик (Cron).** Фоновое регламентное задание запускается каждые 5 минут и забирает все необработанные записи из регистра Очередь_Fast_Track.

- **Шаг 3: Дедупликация (Защита от спама).** 1С "схлопывает" дубли. Если за 5 минут по одному \[GUID + Склад_ID\] прошло 20 продаж, в батч пойдет только **одна** финальная запись (самая свежая) для расчета. Это критически экономит ресурсы 1С и Битрикса.

- **Шаг 4: Вычисление "на лету".** Для отфильтрованного списка 1С запрашивает актуальные цены и высчитывает чистый остаток (логика расчета ниже).

- **Шаг 5: Формирование и Пуш.** 1С пакует данные в JSON-массив и делает POST-запрос на кастомный REST API /update_fast_track Битрикса.

- **Шаг 6: Обработка ответа (Идемпотентность).** \* Если Битрикс возвращает HTTP 200 OK — 1С помечает записи в очереди как "Обработан = Да" (или удаляет их).

  - Если Битрикс возвращает HTTP 5xx/4xx или отвалился по таймауту — записи остаются в очереди. В следующий 5-минутный цикл 1С снова попытается их отправить (с учетом новой дедупликации). Ошибка пишется в журнал 1С.

#### **2. Описание параметров и вычислений**

Все вычисления происходят на стороне 1С в момент сбора батча (Шаг 4).

- GUID (String) — Уникальный идентификатор товара в 1С. Primary Key для матчинга на сайте.

- Store_ID (String) — Уникальный GUID или ID конкретного склада (например, "Беловодск"), на котором произошло изменение.

- Stock_Free (Integer) — **Вычисляемый параметр.** \* *Формула в 1С:* MAX(0, Stock_Physical - Stock_Reserved)

  - *Пояснение:* 1С берет физический остаток на складе Store_ID, вычитает из него товары, зарезервированные документами "Заказ покупателя" на этом же складе. Если из-за пересортицы резерв больше наличия — принудительно отдает 0 (защита от минусов по Эпику 36).

- Price_Retail (Float) — Розничная цена (Тип цены: "Базовая/Розница").

- Price_B2B (Float) — Оптовая цена для прорабов (Тип цены: "Опт/B2B").

- Price_Ecom (Float) — Базовая цена для интернет-магазина (Тип цены: "E-com").

- Price_Promo (Float) — Акционная цена для интернета (Тип цены: "E-com Акция"). Если активной скидки на товар нет, передается 0 или null.

#### **3. Структура JSON-батча (Data Contract)**

Пакет передается плоским массивом объектов. Если товар изменился на двух складах одновременно, он будет передан двумя разными объектами (что упрощает парсинг дельт на стороне Битрикса).

JSON

\[

{

"GUID": "e2a3b1f4-5c6d-7e8f-9a0b-1c2d3e4f5a6b",

"Store_ID": "store_belovodsk_01",

"Stock_Free": 14,

"Price_Retail": 1200.00,

"Price_B2B": 1050.00,

"Price_Ecom": 1150.00,

"Price_Promo": 990.00

},

{

"GUID": "e2a3b1f4-5c6d-7e8f-9a0b-1c2d3e4f5a6b",

"Store_ID": "store_kokzhar_02",

"Stock_Free": 0,

"Price_Retail": 1200.00,

"Price_B2B": 1050.00,

"Price_Ecom": 1150.00,

"Price_Promo": 990.00

}

\]

#### **4. Как Битрикс обрабатывает этот пакет (AC для PHP-разработчика)**

- **Дано:** Битрикс получает POST-запрос с JSON.

- **Когда:** Скрипт принимает данные.

- **Тогда:** 1. Проверяет наличие GUID в своей базе (если товара нет или он деактивирован статусом Toxic_Online — просто **игнорирует** этот объект и идет дальше). 2. Обновляет значение количества товара на конкретном складе Store_ID. 3. Пересчитывает общий остаток для витрины: Total_Stock = Σ (Store_IDs). 4. Обновляет типы цен. Если Price_Promo \> 0, Битрикс устанавливает её как активную (со скидкой), а Price_Ecom делает зачеркнутой старой ценой. Если Price_Promo == 0, выводится только Price_Ecom. 5. Возвращает HTTP 200.
