# **Cross-sell & Bundle (Комплектация)**

**Цель клиента:** Не забыть расходники — клей, затирку, крестики — не дробя заказ и не переходя по карточкам.

## **Эпик 8: Добавление товара в корзину (Фильтрация по складу)**

**Актор:** Клиент \[CMS\]

**US:** Как бизнес, я хочу рекомендовать только то, что физически лежит на том же складе, чтобы не дробить заказ клиента.

**Data Contract:**

- Фильтрация — на стороне фронтенда \[CMS\] перед рендером блока

- Источник остатков: Fast Track → Stock_Free по \[GUID + Store_ID\]

- Активный склад клиента: window.activeStoreId (DEC-22, DEC-23)

- Cross_Sell_GUIDs для каждого товара — из bitrix_catalog.csv (Slow Track, BI-пайплайн)

**Изменения vs оригинал:**

- ✎ Уточнён источник "активного склада": window.activeStoreId (DEC-22), не абстрактный "Store_ID клиента"

- ↑ Добавлен AC-8.3: фильтрация Toxic_Online / ACTIVE=N товаров из блока (EDGE-09 — был в реестре, в оригинальной USM отсутствовал)

- ↑ Явно зафиксирован источник Cross_Sell_GUIDs: Slow Track / BI-пайплайн (не Fast Track)

## **Эпик 9: Умные рекомендации (Market Basket)**

**Актор:** Клиент \[CMS\]

**US:** Как клиент, я хочу видеть подходящие сопутствующие товары, чтобы не забыть купить расходники.

**Data Contract:**

- BI-пайплайн: алгоритм Apriori, 90-дневное окно транзакций (DEC-24)

- Источник: Transactions.csv (Slow Track, T-1)

- Выход BI: Cross_Sell_GUIDs (массив GUID, до 3 позиций) → bitrix_catalog.csv → Битрикс как свойство инфоблока

- Cross_Sell_Source (enum: apriori \| category_fallback) фиксируется в Data Mart BI — прозрачен только для аналитики, клиент UI не видит различия (DEC-24)

- MVP стартует на category_fallback; переключение на apriori — после ≥ 90 дней истории транзакций (DEC-24)

## **Эпик 10: Пакетное добавление в корзину (Bundle)**

**Актор:** Клиент \[CMS\]

**US:** Как клиент, я хочу в один клик добавить основной товар и все выбранные сопутствующие, не переходя по карточкам.

**Data Contract:**

- Полностью фронтенд \[CMS\]: JS-скрипт в шаблоне Аспро

- Количество сопутствующих товаров по умолчанию = 1 единица (Unit_Base, CON-05)

- Ограничение MVP: количество единиц сопутствующего товара = 1; автоматический расчёт расходников — post-MVP (CON-05)

- Шаг инпута сопутствующего товара определяется его Unit_Base (аналогично основному товару, Эпик 4)
