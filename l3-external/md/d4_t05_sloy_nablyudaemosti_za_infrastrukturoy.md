# INF-MON-01 — Слой наблюдаемости \[INFRASTRUCTURE\]

**Версия:** 1.1  
**Ответственный за реализацию:** Bitrix-подрядчик (CMS + CRM)

## Назначение

Определяет минимально необходимый набор мониторинговых агентов, алертов и задач обслуживания, которые должны быть реализованы в рамках Bitrix CMS и Bitrix24 CRM. Без этого блока первые производственные инциденты будут обнаружены клиентами, а не системой.

**Скоуп:** только события на стороне Bitrix.  
**Не входит:** мониторинг 1С-сервисов (ответственность List.kg), BI-пайплайна (ответственность Data Engineer), инфраструктуры серверов (ответственность DevOps).

## Реализация по событиям

### MON-01 — Fast Track батч не получен

**Механизм:** CMS хранит last_fast_track_at DATETIME — метку времени последнего успешно применённого батча (обновляется в конце каждого Fast Track handler'а). CMS-крон запускается каждые 5 мин.

**Логика крона:**

если NOW() - last_fast_track_at \> 15 мин:

если алерт MON-01 ещё не отправлен за последние 60 мин:

отправить Telegram + email

записать last_mon01_alert_at = NOW()

**Таблица метаданных Fast Track** (добавить в DDL INF-CMS-01):

CREATE TABLE b_dom_ft_meta (

id INT PRIMARY KEY DEFAULT 1,

last_ok_at DATETIME NOT NULL,

last_err_at DATETIME,

batch_count INT UNSIGNED DEFAULT 0,

last_slow_track_date DATE,

last_slow_track_ok TINYINT(1) DEFAULT 0

);

Инициализируется при деплое. Обновляется в конце каждого успешного обработчика Fast Track.

**Содержание алерта:**

\[ДОМ ПРОРАБА\] Fast Track — нет данных

Последний успешный батч: {last_ok_at} ({N} мин назад)

Витрина показывает устаревшие остатки.

Требуется проверка: 1С-сервис доступен?

### MON-02 — DLQ вебхуков 1С не пуст

**Механизм:** CRM-агент (scheduled, каждые 10 мин) проверяет таблицу b_dom_processed_1c_events на наличие записей со статусом dlq.

> **Требование к DDL \[INF-CK-02\]:** таблица b_dom_processed_1c_events (определена в Checkout TZ) должна содержать колонку status ENUM('ok', 'dlq') DEFAULT 'ok'. Если в текущем DDL колонки нет — добавить миграцией:
>
> ALTER TABLE b_dom_processed_1c_events
>
> ADD COLUMN status ENUM('ok', 'dlq') DEFAULT 'ok';
>
> Обработчик вебхуков 1С: при исчерпании 3 попыток × 30 сек \[COMMIT-24\] — обновить запись: UPDATE ... SET status = 'dlq'.

**Логика агента:**

dlq_records = SELECT \* FROM b_dom_processed_1c_events

WHERE status = 'dlq'

AND received_at \< NOW() - INTERVAL 30 MINUTE

если dlq_records не пуст:

для каждой записи — создать CRM-задачу менеджеру

(одна задача на запись, с group_by по CRM_Deal_ID)

**Шаблон CRM-задачи:**

Заголовок: ⚠️ Вебхук 1С не доставлен — Сделка \#{CRM_Deal_ID}

Описание:

Event: {event_type}

Получен: {received_at}

Ошибка: {last_error}

Действие: Открыть карточку сделки → кнопка «Переотправить вебхук»

Ответственный: \[Менеджер по умолчанию\]

Дедлайн: +2 часа от создания

### MON-03 — /close_order не закрылся после 3 ретраев

**Механизм:** реализован как часть обработчика ретраев (см. LM TZ Эпик 25.1, патч LM-P4 \[COM-R3-04\]). 3 ретрая с интервалом 5 мин; каждая ошибка логируется. Срабатывает синхронно с переводом сделки в «Требует ручного закрытия» после 3-й неудачи.

**Логика:**

попытка 1 → ошибка 500 → ждать 5 мин

попытка 2 → ошибка 500 → ждать 5 мин

попытка 3 → ошибка 500:

сделка → «Требует ручного закрытия»

создать CRM-алерт (высокий приоритет) → менеджер

отправить Telegram → технический

**Шаблон CRM-алерта:**

🔴 Требует ручного закрытия — Сделка \#{CRM_Deal_ID}

3 попытки /close_order завершились ошибкой.

Последний ответ 1С: {last_1c_response}

Время последней попытки: {last_attempt_at}

Действие: связаться с List.kg, закрыть вручную.

### MON-04 — Slow Track импорт не завершён к 04:00

**Механизм:** CMS-крон запускается в 04:15 (один раз в сутки). Проверяет флаг завершения ночного импорта.

**Slow Track handler** (добавить в конец обработчика импорта):

UPDATE b_dom_ft_meta

SET last_slow_track_date = CURDATE(),

last_slow_track_ok = 1

WHERE id = 1;

**Логика крона 04:15:**

если b_dom_ft_meta.last_slow_track_date != CURDATE()

ИЛИ b_dom_ft_meta.last_slow_track_ok = 0:

отправить email → технический

**Содержание алерта:**

\[ДОМ ПРОРАБА\] Slow Track — импорт не завершён

Дата: {today}

Ожидалось завершение к 04:00.

Витрина работает на данных предыдущего дня.

Проверить: FTP-доступ, формат CSV, лог импортера.

### MON-05 — Заявка Яндекс зависла

**Механизм:** CRM-агент (каждые 30 мин) проверяет сделки в статусе «Передан курьеру» с UF_YANDEX_CLAIM_ID заполненным.

**Логика:**

зависшие = Сделки, где:

статус = «Передан курьеру»

AND updated_at \< NOW() - INTERVAL 4 HOUR

AND UF_YANDEX_CLAIM_ID IS NOT NULL

для каждой сделки:

создать CRM-задачу менеджеру

> **Примечание по таймингу:** updated_at обновляется при переходе в «Передан курьеру» (вебхук driver_assigned). Поле driver_assigned_at как отдельная колонка в сделке не определено — таймер ведётся по updated_at. Точность: ±30 мин (интервал агента).

**Шаблон CRM-задачи:**

Заголовок: ⚠️ Доставка зависла — Сделка \#{CRM_Deal_ID}

Описание:

Claim ID: {UF_YANDEX_CLAIM_ID}

Статус передан курьеру с: {updated_at}

Прошло: {N} ч

Действие: Проверить статус в кабинете Яндекс.GO

или позвонить клиенту.

### MON-06 — Сделка в OOS-статусе без действий

**Механизм:** CRM-агент (каждый час) проверяет сделки в OOS-статусах без движения \> 24 ч.

> **Два разных OOS-статуса** (Processing TZ v1.1):

- «**Требует уточнения (Нехватка)**» — возникает при ответе /create_order: partial (Эпик 17.1)

- «**Уточнение состава**» — возникает при вебхуке oos_at_picking от 1С (Эпик 19.1)

> Агент мониторит оба.

**Логика:**

зависшие = Сделки, где:

статус IN («Требует уточнения (Нехватка)», «Уточнение состава»)

AND updated_at \< NOW() - INTERVAL 24 HOUR

для каждой сделки:

создать CRM-задачу менеджеру (эскалация)

**Шаблон CRM-задачи:**

Заголовок: ⏰ OOS не урегулирован 24ч — Сделка \#{CRM_Deal_ID}

Описание:

Статус: {текущий статус сделки}

Позиции OOS: {UF_OOS_ITEMS}

Клиент ждёт согласования.

Действие: связаться с клиентом, зафиксировать решение.

**AC:**

- <img src="media/image34.png" style="width:0.18056in;height:0.18056in" alt="снят" />MON-06.1: Агент запускается раз в час
