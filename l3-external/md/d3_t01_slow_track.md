# Data Contract: Slow Track

**Версия:** 1.1 **Консьюмер:** Google Cloud Run (BI-пайплайн)

## Общие параметры Slow Track

| **Параметр**        | **Значение**                                                                                                                       |
|---------------------|------------------------------------------------------------------------------------------------------------------------------------|
| Механизм доставки   | FTP-push из 1С:УТ. Режим: пассивный FTP (PASV). Учётная запись: 1c_writer (write-only)                                             |
| Расписание          | Регламентное задание 1С, запуск ежедневно в 02:00                                                                                  |
| Путь на FTP         | /slow_track/YYYY-MM-DD/                                                                                                            |
| Сигнал готовности   | Файл \_DONE.flag — появляется строго после успешной записи **всех** файлов пакета. Cloud Run не обрабатывает пакет без этого флага |
| Хранение на FTP     | 7 дней. Папки старше 7 дней удаляются автоматически после записи флага                                                             |
| Кодировка           | UTF-8 с BOM (EF BB BF) для всех файлов пакета                                                                                      |
| Разделитель полей   | ; (точка с запятой). Строковые поля, содержащие ; или " — в двойных кавычках. Внутренние " экранируются удвоением ""               |
| Числовой формат     | Десятичный разделитель — точка. Без разделителей тысяч                                                                             |
| FTP-доступ (чтение) | Учётная запись: bi_reader (read-only). Учётные данные — у Data Engineer                                                            |
| Файлы пакета        | Catalog.csv, Stocks_By_Store.csv, Transactions.csv, Movements.csv, Picking_Gaps.csv                                                |

## Таблица 1: Catalog.csv

Полный снимок активной номенклатуры. Источник данных для витрины, BI-пайплайна и всех межсистемных FK-ссылок.

| **Колонка**      | **Тип**        | **Nullable**           | **Источник** | **Описание**                                                                                                                                                                                                  |
|------------------|----------------|------------------------|--------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| GUID             | String         | NOT NULL               | 1С           | Внутренний GUID номенклатуры. Формат: uuid4 без фигурных скобок. Primary Key                                                                                                                                  |
| Raw_Name         | String         | NOT NULL               | 1С           | Сырое наименование из 1С без изменений                                                                                                                                                                        |
| Category_L1      | String         | NULL                   | 1С           | Группа номенклатуры верхнего уровня (прямой потомок корня дерева групп)                                                                                                                                       |
| Category_L2      | String         | NULL                   | 1С           | Группа 2-го уровня. Если номенклатура в группе 1-го уровня — совпадает с Category_L1                                                                                                                          |
| Barcode          | String         | NULL (пустая строка)   | 1С           | EAN-13 (приоритет) или внутренний штрихкод. Если нет — пустая строка (не null)                                                                                                                                |
| Supplier_Name    | String         | NULL (пустая строка)   | 1С           | Наименование основного поставщика. Если не задан — пустая строка                                                                                                                                              |
| Cost_Price       | Float          | NULL                   | 1С           | Себестоимость (цена закупки). Десятичная точка, 2 знака                                                                                                                                                       |
| Price_Retail     | Float          | NULL                   | 1С           | Розничная цена                                                                                                                                                                                                |
| Price_Ecom       | Float          | NULL                   | 1С           | Цена интернет-магазина. Если тип цен «Интернет-магазин» не задан — равна Price_Retail                                                                                                                         |
| Price_B2B        | Float          | NULL                   | 1С           | Оптовая цена для прорабов / B2B                                                                                                                                                                               |
| Price_Promo      | Float          | NULL если нет акции    | 1С           | Акционная цена. Строго null при отсутствии активной акции — никогда не 0, не пустая строка                                                                                                                    |
| Stock_Physical   | Integer        | NOT NULL               | 1С           | Суммарный физический остаток по всем активным складам                                                                                                                                                         |
| Stock_Free       | Integer        | NOT NULL               | 1С           | Суммарный свободный остаток: SUM(MAX(0, Physical_i − Reserved_i)). Никогда не отрицательный                                                                                                                   |
| is_returnable    | Integer (0\|1) | NOT NULL (по умолч. 1) | 1С           | Признак возможности возврата. 1 = возврат разрешён, 0 = невозвратный товар. Источник: реквизит ДопПоле_МожноВернуть. Используется в /create_return (предупредительная проверка)                               |
| Is_Toxic_Offline | Integer (0\|1) | NOT NULL (по умолч. 0) | 1С           | Признак токсичного офлайн-товара (систематически слабые продажи в рознице). Заполняется BI-пайплайном через write-back механизм. Источник: реквизит ДопПоле_ТоксичныйОффлайн. 1С только хранит и экспортирует |

> ℹ **Фильтр выгрузки:** Помечен на удаление = Нет AND Вид номенклатуры = Товар. Услуги, комплекты, внутренние статьи — исключены.
>
> ℹ **Price_Promo:** JSON null (без кавычек) при отсутствии акции — не 0, не пустая строка. Аналогичное правило действует в Fast Track JSON-батче.
>
> ℹ **is_returnable:** значение 0 означает что /create_return вернёт warning в поле warnings\[\], но документ всё равно создаётся (мягкая проверка, CON-15).
>
> ℹ **Is_Toxic_Offline:** поле инициализируется 1С со значением 0. BI-пайплайн обновляет его через write-back — 1С не рассчитывает признак самостоятельно.

## Таблица 2: Stocks_By_Store.csv

Остатки в разрезе склад × товар для активных позиций.

| **Колонка**    | **Тип** | **Nullable** | **Источник** | **Описание**                                                                                                |
|----------------|---------|--------------|--------------|-------------------------------------------------------------------------------------------------------------|
| GUID           | String  | NOT NULL     | 1С           | FK → Catalog.GUID                                                                                           |
| Store_ID       | String  | NOT NULL     | 1С           | Уникальный код склада (реквизит «Код» из справочника Склады). Формат: snake_case, например store_kokzhar_02 |
| Store_Name     | String  | NOT NULL     | 1С           | Читаемое название склада («Кок Жар», «Беловодск»)                                                           |
| Stock_Physical | Integer | NOT NULL     | 1С           | Физический остаток на складе                                                                                |
| Stock_Reserved | Integer | NOT NULL     | 1С           | Зарезервировано активными «Заказами покупателей» на этом складе                                             |
| Stock_Free     | Integer | NOT NULL     | 1С           | MAX(0, Stock_Physical − Stock_Reserved). Никогда не отрицательный                                           |

> ℹ **Фильтр строк:** включаются только записи где Stock_Physical \> 0 OR Stock_Reserved \> 0. Строки с обнулёнными физическим остатком и резервом в файл не попадают.
>
> ℹ **Семантика отсутствующей строки:** если пара \[GUID × Store_ID\] отсутствует в файле — BI-пайплайн интерпретирует Stock_Free = 0 для этой пары. Это консистентно с бизнес-логикой: MAX(0, 0−0) = 0.
>
> ℹ **Store existence:** если все товары на складе обнулились, склад полностью исчезает из файла. Данный файл не является источником истины для измерения «список активных складов». Store-dimension таблицу строить на основе этого файла не рекомендуется — использовать отдельный справочник складов или Store_ID из Transactions.csv.
>
> ℹ **Аномалия Physical = 0, Reserved \> 0:** такая строка попадает в файл (Reserved \> 0). Stock_Free = 0. Строка видна BI для data quality мониторинга (фантомный резерв).

## Таблица 3: Transactions.csv

Все проведённые кассовые и торговые операции за T-1. Источники: Чек ККМ (розница) и Реализация товаров (B2B). Диапазон: 00:00:00 — 23:59:59 предыдущего дня.

| **Колонка**  | **Тип**  | **Nullable**         | **Источник** | **Описание**                                                                                                                                        |
|--------------|----------|----------------------|--------------|-----------------------------------------------------------------------------------------------------------------------------------------------------|
| Date_Time    | DateTime | NOT NULL             | 1С           | Дата и время проведения документа. Формат: YYYY-MM-DD HH:MM:SS (ISO 8601 без T-разделителя)                                                         |
| Receipt_ID   | String   | NOT NULL             | 1С           | Номер документа 1С. Формат: строковый номер в нотации 1С, например «КЧКМ-000456» или «РТ-000123». Не UUID. Уникален в рамках типа документа за день |
| Client_ID    | String   | NULL (пустая строка) | 1С           | GUID контрагента (uuid4). Для розничных ЧекККМ — пустая строка (анонимная продажа)                                                                  |
| Phone        | String   | NULL (пустая строка) | 1С           | Телефон контрагента если заполнен в справочнике. Пустая строка если не идентифицирован                                                              |
| Store_ID     | String   | NOT NULL             | 1С           | Код склада/магазина документа. FK → Store_ID в Stocks_By_Store                                                                                      |
| GUID         | String   | NOT NULL             | 1С           | GUID номенклатуры (uuid4). FK → Catalog.GUID                                                                                                        |
| Qty          | Float    | NOT NULL             | 1С           | Количество. Может быть дробным для весовых товаров                                                                                                  |
| Sum_Base     | Float    | NOT NULL             | 1С           | Сумма по регулярной цене без скидки                                                                                                                 |
| Sum_Discount | Float    | NOT NULL             | 1С           | Сумма скидки. 0 если скидки нет                                                                                                                     |
| Promo_Type   | String   | NULL                 | 1С           | Тип применённой скидки: «Карта лояльности», «Акция», «Ручная». null если скидка не применялась                                                      |
| Sum_Revenue  | Float    | NOT NULL             | 1С           | Итоговая сумма к оплате. Инвариант: Sum_Base − Sum_Discount = Sum_Revenue (допуск ±0.01 на округление)                                              |
| Sum_Cost     | Float    | NULL                 | 1С           | Себестоимость проданного. null если партия не закрыта на момент выгрузки (0 в этом случае — с WARN в журнале 1С)                                    |

> ℹ **Receipt_ID** не является UUID и не должен использоваться как глобально уникальный ключ между разными типами документов. Уникальность: в рамках одного типа документа (ЧекККМ или Реализация) за один день.
>
> ℹ **Sum_Cost:** при незакрытых партиях 1С пишет значение 0 и фиксирует WARN в журнале регистрации. BI-пайплайн должен учитывать что Sum_Cost = 0 не означает нулевую себестоимость.

## Таблица 4: Movements.csv

Все товарные движения за T-1 по Virtual Node Pattern. Диапазон: 00:00:00 — 23:59:59 предыдущего дня.

| **Колонка**   | **Тип**  | **Nullable** | **Источник** | **Описание**                                                                                                                               |
|---------------|----------|--------------|--------------|--------------------------------------------------------------------------------------------------------------------------------------------|
| Date_Time     | DateTime | NOT NULL     | 1С           | Дата и время операции. Формат: YYYY-MM-DD HH:MM:SS                                                                                         |
| Document_Type | String   | NOT NULL     | 1С           | Тип документа-источника (рус.). Значения: Поступление, Реализация, Чек ККМ, Перемещение, Списание, Возврат от покупателя, Инвентаризация   |
| GUID          | String   | NOT NULL     | 1С           | GUID номенклатуры (uuid4). FK → Catalog.GUID                                                                                               |
| Qty           | Float    | NOT NULL     | 1С           | Количество. Всегда положительное                                                                                                           |
| Sum_Total     | Float    | NULL         | 1С           | Сумма движения в ценах закупки/списания                                                                                                    |
| Store_From_ID | String   | NOT NULL     | 1С           | Откуда ушёл товар. NULL запрещён — при отсутствии реального склада использовать виртуальный узел (см. реестр ниже)                         |
| Store_To_ID   | String   | NOT NULL     | 1С           | Куда пришёл товар. NULL запрещён — при отсутствии реального склада использовать виртуальный узел                                           |
| Reason        | String   | NULL         | 1С           | Причина/описание движения. Для Store_To_ID = \_WRITEOFF: строго один из трёх enum-значений (см. ниже). Для всех остальных: свободный текст |

### Реестр виртуальных узлов (Virtual Node Pattern)

Поля Store_From_ID и Store_To_ID никогда не содержат NULL. При отсутствии реального склада-источника или склада-назначения используется один из виртуальных узлов:

| **Store_ID** | **Смысл**                                                                   |
|--------------|-----------------------------------------------------------------------------|
| \_SUPPLIER   | Поставщик — источник прихода товара                                         |
| \_CUSTOMER   | Покупатель — приёмник при продаже (Store_To_ID)                             |
| \_RETURN_IN  | Покупатель как источник возврата (Store_From_ID при возврате от покупателя) |
| \_WRITEOFF   | Списание: брак, бой, недостача                                              |
| \_ADJUST     | Инвентаризационная корректировка (излишки и недостачи)                      |

### Маппинг Document_Type → виртуальные узлы

| **Document_Type**           | **Store_From_ID** | **Store_To_ID**      |
|-----------------------------|-------------------|----------------------|
| Поступление (от поставщика) | \_SUPPLIER        | store\_\* (реальный) |
| Реализация / Чек ККМ        | store\_\*         | \_CUSTOMER           |
| Перемещение                 | store\_\*         | store\_\*            |
| Списание                    | store\_\*         | \_WRITEOFF           |
| Возврат от покупателя       | \_RETURN_IN       | store\_\*            |
| Инвентаризация (излишек)    | \_ADJUST          | store\_\*            |
| Инвентаризация (недостача)  | store\_\*         | \_ADJUST             |

### Поле Reason: enum для Store_To_ID = \_WRITEOFF

Для строк где Store_To_ID = \_WRITEOFF поле Reason обязательно и должно содержать одно из трёх значений. Документ «Списание товаров» в 1С не проводится без выбора причины из справочника (задача 1С-CL-01).

| **Значение Reason** | **Описание**                                    | **Кто инициирует**        |
|---------------------|-------------------------------------------------|---------------------------|
| delivery_damage     | Порча при доставке (Яндекс и другие курьерские) | Рекламация клиента        |
| warehouse_damage    | Внутрискладской бой                             | Кладовщик при обнаружении |
| manufacturer_defect | Брак производителя                              | Рекламация клиента        |

> ℹ Для всех остальных типов движений (Store_To_ID ≠ \_WRITEOFF) поле Reason — свободный текст без ограничений (формат: «{НомерДокумента} {КомментарийДокумента}»). NULL допустим.
>
> ℹ BI-пайплайн использует Reason как измерение в блоке «Убытки от брака» и расчёте Defect_Rate по номенклатуре.
>
> ℹ **Legacy-данные** (до внедрения типизации 1С-CL-01): строки с нетипизированной причиной выгружаются с исходным текстом + WARN в журнале 1С. BI-пайплайн рекомендуется тегировать такие строки как Writeoff_Reason = "UNKNOWN".

## Таблица 5: Picking_Gaps.csv

Расхождения между зарезервированным и фактически найденным количеством при сборке заказа. Источник: регистр накопления Расхождения_При_Сборке (1С:УТ) — заполняется при переводе «Заказа покупателя» в статус "Нехватка_При_Сборке" (задача 1С-PR-04).

| **Колонка** | **Тип**        | **Nullable** | **Источник**           | **Описание**                                                               |
|-------------|----------------|--------------|------------------------|----------------------------------------------------------------------------|
| DateTime    | DateTime (ISO) | NOT NULL     | Расхождения_При_Сборке | Момент фиксации нехватки кладовщиком                                       |
| Order_ID    | VARCHAR(50)    | NOT NULL     | Расхождения_При_Сборке | Номер «Заказа покупателя» в 1С (строковый номер документа)                 |
| CRM_Deal_ID | VARCHAR(50)    | NOT NULL     | Расхождения_При_Сборке | ID связанной сделки Битрикс24. FK → CRM_Deal_ID                            |
| GUID        | VARCHAR(36)    | NOT NULL     | Расхождения_При_Сборке | GUID номенклатуры (uuid4). FK → Catalog.GUID                               |
| Store_ID    | VARCHAR(50)    | NOT NULL     | Расхождения_При_Сборке | Код склада сборки                                                          |
| Qty_Missing | DECIMAL(15,3)  | NOT NULL     | Расхождения_При_Сборке | Количество единиц которых не хватило при физической сборке                 |
| Qty_Ordered | DECIMAL(15,3)  | NOT NULL     | Расхождения_При_Сборке | Количество единиц в исходном заказе (контекст для процентного расхождения) |

> ℹ **Периодичность:** T-1, в составе ночной выгрузки Slow Track. Путь: /slow_track/YYYY-MM-DD/Picking_Gaps.csv
>
> ℹ **Пустой файл:** если расхождений за день не было — файл формируется с заголовком и пустым телом (0 строк данных). Файл **не отсутствует** — его наличие без \_DONE.flag не сигнализирует о готовности пакета.
>
> ℹ **Применение в BI:** выявление складов с систематическими расхождениями, оценка частоты недостач, эскалация при превышении пороговых значений.
