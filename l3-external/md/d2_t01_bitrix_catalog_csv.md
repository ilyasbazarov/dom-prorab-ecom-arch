# Data Contract: bitrix_catalog.csv

**Версия:** 2.0  
**Статус:** Готово к реализации  
**Источник (Producer):** BI-пайплайн — Google Cloud Run (Data Engineer)  
**Потребитель (Consumer):** Битрикс CMS \[CMS\] (подрядчик по Битриксу)  
**Назначение:** Передача обогащённых и нормализованных данных каталога из BI-пайплайна на витрину. Является слоем между сырым Catalog.csv (1С → BI) и инфоблоком Битрикс.

## Механизм доставки: FTP-pull

| **Параметр**              | **Значение**                                                                             |
|---------------------------|------------------------------------------------------------------------------------------|
| FTP-сервер                | Тот же сервер что и Slow Track                                                           |
| Путь записи (BI)          | /bi_catalog/YYYY-MM-DD/bitrix_catalog.csv                                                |
| Сигнал готовности         | /bi_catalog/YYYY-MM-DD/\_DONE.flag — записывается строго **после** успешной записи файла |
| FTP-аккаунт (запись, BI)  | bi_writer — write-only, путь /bi_catalog/                                                |
| FTP-аккаунт (чтение, CMS) | cms_reader — read-only, путь /bi_catalog/                                                |
| Расписание CMS            | Опрос каждые 30 мин с 04:00. При \_DONE.flag — немедленный импорт                        |
| Хранение на FTP           | 7 дней                                                                                   |
| Atomicity                 | BI пишет во временное имя → переименовывает → пишет \_DONE.flag                          |
| При ошибке импорта CMS    | Логировать, повторить на следующем цикле. Не удалять флаг                                |

## Схема: Блок B — Нормализованные названия, статус и спрос (BI-пайплайн)

| **Колонка** | **Тип** | **Nullable**         | **Источник**                                              | **Описание**                                                                                                                                                                        |
|-------------|---------|----------------------|-----------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Name_Clean  | String  | NOT NULL             | BI (Regex/LLM из Raw_Name)                                | **Отображаемое название на витрине.** Если BI не сформировал — дублируется Raw_Name, Action_Flag = Manual_Rename                                                                    |
| Ecom_Status | Enum    | NOT NULL             | BI (Action_List)                                          | Статус товара для e-com. Управляет ACTIVE в инфоблоке Битрикс. Значения: Stand_Alone, Add_On_Only, Toxic_Online, Review_Required                                                    |
| Action_Flag | String  | NULL (пустая строка) | BI                                                        | Служебный флаг для команды e-com. Видно в **админке**, не на витрине. Значения: Manual_Rename, Ratio_Error, UNKNOWN_SUPPLIER                                                        |
| Sales_30d   | Integer | NULL                 | BI (агрегация из Transactions.csv, rolling 30-day window) | Количество проданных единиц GUID за последние 30 дней. Используется для сортировки fallback cross-sell: Sales_30d DESC. При Sales_30d = 0 — товар в конце списка. **\[COM-CK-05\]** |

### Маппинг Ecom_Status → ACTIVE \[COM-CK-02\]

| **Ecom_Status** | **ACTIVE** | **Поведение на витрине**                                                                                                                                 | **Поведение в cross-sell/bundle**                                    |
|-----------------|------------|----------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------|
| Stand_Alone     | **Y**      | Полноценная карточка, кнопка «В корзину» активна                                                                                                         | Отображается                                                         |
| Add_On_Only     | **Y**      | Карточка есть. Кнопка «В корзину» **заблокирована** при самостоятельном добавлении. Сообщение: «Этот товар доступен только в комплекте с другим товаром» | Отображается — cross-sell/bundle загружает товар явным GUID-запросом |
| Toxic_Online    | **N**      | Скрыт из каталога и поиска. Fast Track батч игнорируется (EDGE-04)                                                                                       | Не отображается                                                      |
| Review_Required | **N**      | Скрыт до ручного перевода в Stand_Alone администратором                                                                                                  | Не отображается                                                      |

> **Правило приоритета ACTIVE:**

1.  Is_Toxic_Offline = 1 → ACTIVE=N **принудительно**, независимо от Ecom_Status

2.  Ecom_Status применяется если Is_Toxic_Offline = 0

3.  Stock_Free = 0 (из Fast Track) ≠ ACTIVE=N. Карточка с нулевым остатком остаётся ACTIVE=Y → «Под заказ» + «Сообщить о поступлении»

> **AC для подрядчика \[CMS\] — Add_On_Only \[COM-CK-02\]:**

- Блокировка в корзине реализуется проверкой Ecom_Status = Add_On_Only при добавлении позиции

- Cross-sell/bundle компонент загружает Add_On_Only-товары явным GUID-запросом, **игнорируя фильтр ACTIVE**

- Фильтр EDGE-09 (AC 8.3) исключает из cross-sell только Ecom_Status IN (Toxic_Online, Review_Required). Add_On_Only — **не исключается**

## Схема: Блок C — Единицы измерения и физические параметры (BI-пайплайн)

| **Колонка**   | **Тип** | **Nullable**         | **Источник**                                | **Описание**                                                                                                                                                                  |
|---------------|---------|----------------------|---------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Unit_Base     | String  | NULL (пустая строка) | BI (Action List → Regex → Category Map)     | Базовая единица продажи (шт, м², пог.м, упак и т.д.). Шаг инпута в корзине = 1 Unit_Base                                                                                      |
| Unit_Alt      | String  | NULL (пустая строка) | BI                                          | Альтернативная единица. Если пустая — переключатель единиц не выводится                                                                                                       |
| Ratio         | Float   | NULL                 | BI                                          | Коэффициент Unit_Alt → Unit_Base. Ratio ≤ 0 или NULL → переключатель не выводится, Action_Flag = Ratio_Error. CMS вычисляет Price_m2 = Price_Ecom / Ratio клиентским скриптом |
| Weight_kg     | Float   | NULL                 | BI (Action List → Regex → Category Default) | Вес одной Unit_Base в кг. Для расчёта доставки                                                                                                                                |
| Volume_m3     | Float   | NULL                 | BI                                          | Объём одной Unit_Base в м³. Для определения типа кузова совместно с is_oversize                                                                                               |
| Weight_Source | Enum    | NULL (пустая строка) | BI                                          | Источник Weight_kg / Volume_m3: action_list \| regex \| category_default. Информационное поле для Data Engineer, CMS не использует                                            |

Нормализован через Category Mapping Sheet (DEC-18)                                                                                                                                                                                 |
| Attr\_\* (полный список) | String  | NULL (пустая строка) | BI (Regex/LLM + Category Map) | **⚠️ OQ-BC-03a: полный список финализирует Data Engineer.** Фиксированный набор колонок по всем категориям; пустая строка для неприменимых. Нормализация на стороне BI. CMS не нормализует. При импорте: пустая строка → свойство не заполняется |

## Схема: Блок E — Cross-sell (BI Data Mart)

| **Колонка**       | **Тип**             | **Nullable**              | **Источник**   | **Описание**                                                                                         |
|-------------------|---------------------|---------------------------|----------------|------------------------------------------------------------------------------------------------------|
| Cross_Sell_GUIDs  | String (JSON array) | NULL (пустой массив \[\]) | BI (Data Mart) | JSON-массив GUID связанных товаров. Формат: \["uuid4","uuid4",...\]. Хранится как свойство инфоблока |
| Cross_Sell_Source | Enum                | NULL (пустая строка)      | BI             | category_fallback (MVP) \| apriori (включается при ≥90 дней истории транзакций, DEC-24)              |

> **AC для подрядчика \[CMS\] — Cross-sell рендер:**

- При Cross_Sell_GUIDs пуст → fallback: топ-3 из Category_L2 по Sales_30d DESC (при равном — вторично по Stock_Free DESC), GUID != текущий, Ecom_Status NOT IN (Toxic_Online, Review_Required), Stock_Free\[window.activeStoreId\] \> 0 \[COM-CK-03, COM-CK-05\]

- Загрузка по GUID-запросу игнорирует фильтр ACTIVE — чтобы Add_On_Only (ACTIVE=Y) отображался \[COM-CK-02\]

- Фильтр по window.activeStoreId: показывать только товары с Stock_Free \> 0 на активном складе (CON-05: кол-во по умолчанию = 1 Unit_Base)
